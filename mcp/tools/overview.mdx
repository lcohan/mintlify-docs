The Alphacast MCP server exposes 22 tools across four functional areas. Use this page as a high-level map; click any tool name to jump to its full input schema and example prompt on the corresponding detail page.

## Safety boundaries

<Warning>
  The server intentionally does **not** expose:

  - `delete_pipeline` — deleting an entire pipeline is irreversible from the API. Delete pipelines from the Alphacast web UI when needed.
  - Repository or dataset deletion — same reasoning. Use the web UI for irreversible operations.
  - `run_from_step_id` / `runFromStepId` — this parameter exists in older parts of the REST API but is not supported. Do not use it; it will be ignored or rejected.
</Warning>

## Tool conventions

- **IDs** are always numeric (`integer`). Tools accept strings and coerce them — `"42"` works the same as `42`.
- **Formats** for downloads: `json` (default for `download_dataset`) or `csv`.
- **Auto-run** behavior: any pipeline write tool (`create_pipeline`, `add_pipeline_step`, `edit_pipeline_step`, `delete_pipeline_step`) immediately validates the resulting pipeline and queues a run. There is no separate "run" tool to call afterwards.
- **Step previews** (`get_step_preview`) poll up to 60 seconds. If the run is slow, the tool returns a timeout error — retry or use `get_step_data` to read cached output instead.
- **Permissions** mirror the REST API. The API key's permission on each repository determines which tools succeed.

## Underlying API

Each tool maps to one or more endpoints in the Alphacast REST API. The mapping is documented on each tool's detail page. You don't need to call those endpoints yourself — the MCP server is the recommended interface for programmatic access.
