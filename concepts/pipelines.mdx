A pipeline is an ordered list of steps that process data from start to finish. You configure each step — what type it is, what order it runs in, and how it behaves — and Alphacast executes the pipeline automatically whenever a step changes. Pipelines live inside repositories alongside the datasets they produce.

## How pipelines work

Each step in a pipeline has three required fields:

- **`type`** — what the step does (e.g., fetch data, filter rows, publish results).
- **`order`** — the position of the step in the sequence (integer, starting from `0`).
- **`configuration`** — a JSON object with step-specific settings.

When you add or modify a step, Alphacast automatically cancels any in-progress run for that pipeline and starts a fresh one. You can also trigger runs manually, which is useful for scheduled jobs or after uploading new source data.

## Step types

Alphacast supports six categories of steps — fetch, transform, calculation, merge & reshape, AI, and publish. Each category covers several concrete step types with their own configuration schemas.

## Building pipelines programmatically

The Alphacast MCP server exposes pipeline operations as tools that any MCP-compatible client (Claude Desktop, Claude Code, Cursor, and others) can invoke directly. This is the recommended way to build, edit, and run pipelines from code or from a chat agent.
