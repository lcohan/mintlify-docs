---
title: "GET /pipelines/{id} — Get a Pipeline"
sidebarTitle: "Get"
description: "Retrieve a single pipeline by its numeric ID or URL slug. Returns the full pipeline including all steps, configuration, and repository assignment."
---

Use this endpoint to inspect a specific pipeline. You can identify the pipeline by its numeric `id` or by its human-readable `slug`. The response includes the complete step list and all configuration fields — the same structure returned by `GET /pipelines`.

## Authentication

All requests require HTTP Basic Auth. Pass your API key as the username and leave the password blank.

## Path parameters

<ParamField path="pipeline_id_or_slug" type="string" required>
  The pipeline's numeric ID (e.g. `42`) or its URL slug (e.g. `cpi-inflation-monitor`). Both forms are accepted.
</ParamField>

## Response

<ResponseField name="id" type="integer">
  Unique numeric identifier for the pipeline.
</ResponseField>

<ResponseField name="name" type="string">
  Human-readable display name of the pipeline.
</ResponseField>

<ResponseField name="repositoryId" type="integer">
  ID of the repository this pipeline belongs to.
</ResponseField>

<ResponseField name="slug" type="string">
  URL-friendly identifier. Can be used interchangeably with the numeric `id` across the Pipelines API.
</ResponseField>

<ResponseField name="runtimeVersion" type="string">
  Runtime environment version used during pipeline execution.
</ResponseField>

<ResponseField name="steps" type="array">
  Ordered list of step objects that define the pipeline's logic.

  <Expandable title="step properties">
    <ResponseField name="id" type="integer">
      Server-assigned step identifier.
    </ResponseField>
    <ResponseField name="type" type="string">
      Step type. See the [steps reference](/api/pipelines/steps) for all supported types.
    </ResponseField>
    <ResponseField name="order" type="integer">
      Zero-based position of the step in the execution sequence.
    </ResponseField>
    <ResponseField name="configuration" type="object">
      Step-specific configuration. Structure varies by `type`.
    </ResponseField>
  </Expandable>
</ResponseField>

## Example

<RequestExample>

```bash cURL (by ID)
curl https://api.alphacast.io/pipelines/42 \
  --user YOUR_API_KEY:
```

```bash cURL (by slug)
curl https://api.alphacast.io/pipelines/cpi-inflation-monitor \
  --user YOUR_API_KEY:
```

```python Python
import requests

# Fetch by numeric ID
response = requests.get(
    "https://api.alphacast.io/pipelines/42",
    auth=("YOUR_API_KEY", ""),
)

# Or by slug
response = requests.get(
    "https://api.alphacast.io/pipelines/cpi-inflation-monitor",
    auth=("YOUR_API_KEY", ""),
)

pipeline = response.json()
```

```javascript Node.js
const response = await fetch("https://api.alphacast.io/pipelines/42", {
  headers: {
    Authorization: "Basic " + btoa("YOUR_API_KEY:"),
  },
});
const pipeline = await response.json();
```

</RequestExample>

<Tip>
  Store the `slug` alongside your pipeline ID if you want stable, readable references in scripts and URLs. Slugs do not change when you rename a pipeline.
</Tip>
